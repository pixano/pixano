# Visual Question Answering

Annotate images with question-answer conversations. This guide uses the [VQAv2](https://visualqa.org/) dataset and covers: import, explore the VQA panel UI, ask and answer questions, and connect AI models for answer generation.

## Prerequisites

- Pixano installed (see [Installation](/pixano/getting_started/installing_pixano/))
- A data directory initialized with `pixano init ./my_data`

## Import the dataset

### Generate the sample

The repository includes a script that downloads a VQAv2 subset from HuggingFace:

```bash
pip install datasets pillow
python examples/vqav2/generate_sample.py ./vqav2_sample --num-samples 50
```

The resulting folder:

```
vqav2_sample/
  validation/
    000000.jpg
    000001.jpg
    ...
    metadata.jsonl
```

Each line in `metadata.jsonl` describes one image with its questions and answers:

```json
{
  "views": { "image": "000000.jpg" },
  "messages": [
    {
      "question": {
        "content": "Where are the kids riding?",
        "question_type": "OPEN"
      },
      "responses": [{ "content": "carnival ride" }]
    }
  ]
}
```

Format details:

- `views.image` points to the image file.
- `messages` is a list of question-answer exchanges.
- Each message has a `question` with `content` and `question_type` (`"OPEN"` for free-form, `"multi_choice"` for multiple choice).
- `responses` is a list of answers, each with `content` and an optional `user` field.

<Callout type="note" title="Multiple-choice questions" collapsible>

For multiple-choice questions, set `question_type` to `"multi_choice"` and add a `choices` list:

```json
{
  "question": {
    "content": "What is in the image?",
    "question_type": "multi_choice",
    "choices": ["a cat", "a dog", "a bird", "a fish"]
  },
  "responses": [{ "content": "a cat" }]
}
```

</Callout>

### Review the dataset info

The dataset info at `examples/vqav2/info.py`:

```python
from pixano.datasets import DatasetInfo
from pixano.datasets.workspaces import WorkspaceType
from pixano.schemas import BBox, CompressedRLE, Entity, Image, Message, Record


class VQAv2Entity(Entity):
    category: str = ""
    subcategory: str = ""


dataset_info = DatasetInfo(
    name="VQAv2 Sample",
    description="Sample import for VQAv2 visual question answering data.",
    workspace=WorkspaceType.IMAGE_VQA,
    record=Record,
    entity=VQAv2Entity,
    message=Message,
    bbox=BBox,
    mask=CompressedRLE,
    views={"image": Image},
)
```

Key elements:

- `WorkspaceType.IMAGE_VQA` enables the VQA conversation panel.
- `Message` is the annotation type that stores question-answer pairs.
- `BBox` and `CompressedRLE` are included because VQAv2 also supports object annotation alongside Q&A.

### Run the import

```bash
pixano data import ./my_data ./vqav2_sample \
    --info examples/vqav2/info.py:dataset_info
```

### Launch the server

```bash
pixano server run ./my_data
```

## Explore in the UI

### Item page — VQA mode

When you open an item from a VQA dataset, you see:

- **Image** in the center.
- **VQA panel** on the left — displaying model settings and a list of questions with answers.

#### VQA model settings

At the top of the VQA panel, the model settings let you connect to a [Pixano Inference](https://pixano.github.io/pixano-inference/latest) server for AI-generated answers:

- The **star icon** shows the connection status: red (not connected), orange (connected but no suitable model), green (model ready).
- A **dropdown** selects the model. The **+** icon instantiates a new model.
- The **gear icon** opens prompt settings and temperature controls.

#### Questions and answers

Below the model settings, you see a list of questions:

- Each question shows a **check icon** (red = no answer, green = answered), the question number, and a **star icon** to generate an AI answer.
- Click a question to expand it and see or edit answers.
- Click **"Add question"** to open the QA Editor where you can type a question or generate one with a connected model.
- Multiple answers per question are supported (e.g. from different annotators).

## Annotate

### Add a question manually

1. Click **"Add question"** in the VQA panel.
2. Choose the question type (open or multiple choice).
3. Type the question.
4. Click confirm.

### Answer a question

1. Expand a question in the list.
2. Type your answer in the response field.
3. You can also click the **star icon** to generate an answer from a connected model and then correct it.

### Object annotation

VQA datasets also support standard annotation tools. Use the bounding box, polygon, or keypoints tools to annotate objects in the image alongside the Q&A workflow.

<Callout type="tip" title="AI-assisted workflow">
Connect a vision-language model via Pixano Inference to generate both questions and answers automatically. Review and correct the AI outputs to build high-quality annotations faster.
</Callout>
